# GithubFundations
Repositório do curso para o curso de preparação para a certificação Github Fundations 
https://www.youtube.com/watch?v=ukyGmIXMfEg&amp;list=PLbwy5IKtFeIGBE5UNg57unH6SMzIeqmKp
O exame foi estruturado para validar se você compreende o funcionamento do ecossistema e sabe utilizar as ferramentas de colaboração no dia a dia como desenvolvedor. Ele é dividido em 4 grandes blocos de conhecimento:

1. Introdução ao Git e GitHub (+- 32% da prova)
Este bloco foca nos conceitos teóricos de controle de versão e no fluxo de trabalho padrão.

Fundamentos do Git: Compreender o que é um sistema de controle de versão distribuído; a diferença entre os comandos locais do Git e a hospedagem em nuvem do GitHub; conceitos de repositórios locais e remotos, commits e hashes de identificação.

O Fluxo do GitHub (GitHub Flow): Dominar o ciclo de vida do código: criar ramificações (branches), commitar alterações, enviar para o servidor, abrir uma solicitação de puxada (Pull Request), debater o código e realizar a fusão (merge).

Estrutura de Contas: Diferenças de escopo, visibilidade e recursos entre contas de usuários individuais, Organizações (Organizations) e Empresas (Enterprises).

Acesso e Autenticação: Como configurar conexões seguras utilizando chaves SSH e HTTPS (com tokens de acesso pessoal - PATs); importância da autenticação de dois fatores (2FA).

2. Trabalhando com Repositórios no GitHub (+- 28% da prova)
Aqui é avaliada a sua capacidade de criar, organizar e gerenciar a documentação de um projeto.

Criação e Gestão de Projetos: Inicialização de repositórios, clonagem, adição de arquivos, uso de arquivos .gitignore para ignorar arquivos temporários e uso de modelos (templates) de repositórios.

Arquivos Obrigatórios de Documentação: Compreender o propósito e a estrutura de arquivos como:

README.md (Apresentação do projeto).

LICENSE (Termos de uso do código).

CONTRIBUTING.md (Regras para outros desenvolvedores colaborarem).

CODEOWNERS (Definição automática de quem deve revisar partes específicas do código).

Sintaxe Markdown: Escrever documentações, formatar tabelas, links, listas e blocos de código usando o padrão Markdown do GitHub (GitHub Flavored Markdown).

Gists e Wikis: Saber quando usar o GitHub Wiki para documentações longas e Gists para compartilhar trechos rápidos de código ou anotações estruturadas.

3. Colaboração em Equipe (+- 25% da prova)
Este módulo cobre as ferramentas que o desenvolvedor usa para se comunicar e resolver problemas com o time.

Gerenciamento de Demandas (Issues): Criação e acompanhamento de tarefas, uso de marcadores (labels), marcos (milestones) para definir metas de entrega e vinculação de tarefas a códigos.

Ciclo de Vida do Pull Request: Revisão de código, comentários em linhas específicas, aprovação de alterações, resolução de conflitos simples de merge e como fechar tarefas automaticamente vinculando um PR a uma Issue (usando palavras-chave como closes ou fixes).

Gerenciamento de Projetos: Utilização básica do GitHub Projects (visões em quadros Kanban, tabelas e cronogramas) para organizar o fluxo de tarefas pessoal ou do time.

Comunicações: Configuração e gerenciamento de notificações, uso do GitHub Discussions para conversas que não geram alteração direta no código e respostas salvas (saved replies) para acelerar a comunicação.

4. Recursos de Produtividade Moderna (+- 15% da prova)
Os novos recursos criados para acelerar a entrega do desenvolvedor direto pela plataforma.

Ambientes Virtuais de Desenvolvimento: Entender o conceito e o funcionamento do GitHub Codespaces (máquinas virtuais na nuvem para codificar direto do navegador ou do VS Code local usando o arquivo devcontainer.json) e a diferença para o editor rápido (github.dev).

Conceitos de Automação (GitHub Actions): Entendimento básico do que são fluxos de trabalho (workflows) em arquivos YAML, componentes básicos (Eventos, Jobs e Steps) e o que é Integração Contínua (CI). Nota: não cai escrita complexa de código de automação, apenas o conceito de uso.

Introdução ao GitHub Copilot: Entender os recursos básicos do assistente de IA, geração de código contextualizado, comandos comuns no chat e noções sobre os planos existentes.

🎯 Formato da Prova: O exame é composto por 75 questões de múltipla escolha, tem duração de 100 minutos e a pontuação mínima para passar costuma rodar na faixa dos 70%.
